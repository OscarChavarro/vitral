USAGE

This applet-based program includes an interaction scheme based on the following mouse and keyboard events, managed by standard Vitral SDK toolkit interaction controllers:
  • Mouse drag with first button: camera rotation
  • Mouse drag with second button: camera advance and roll
  • Mouse drag with third button: camera tile
  • Keyboard keys x, X, y, Y, z and Z: camera position
  • Keyboard arrow keys: camera rotation
  • Keyboard keys s and S: camera roll
  • Keyboard keys a and A: camera angle for field of view
  • Keyboard keys n and N: camera near plane
  • Keyboard keys f and F: camera far plane
  • Keyboard key i: print current camera state to java console

TROUBLESHOOTING

Before searching for reasons of a non working applet, consider that there are two different ways to use a JOGL based applet:
  • Directly calling the application applet class (a stand-alone applet version): this method requires user to previously install JOGL in the java extensions directories (jdk/jre/lib/ext and jre/lib/ext) and native libraries in an operating system directory for libraries (system32 in Windows, or some lib in Unix).
  • Indirect calling via the JOGL applet loader com.sun.opengl.util.JOGLAppletLauncher class (a subapplet version): this doesn't require users to have any JOGL installed, but only works well if JOGL is not installed.
If this applet is not working, can be due to a different set of causes, mostly:
  • If the webserver use a Unix/Linux based operating system, do not use symbolic links, as applet subsystem gives security exceptions
  • Some needed .jar file is not available in the webserver's applet codebase directory, or jogl-natives are not available for this platform
  • The jogl jars included in webserver's applet codebase directory are not the signed versions from Sun Microsystems. Check they where downloaded from http://download.java.net/media/jogl/builds/archive/jsr-231-webstart-current.
  • You are trying to use a subapplet version of this page in an environment where jogl is locally installed in the java extension directory, or from appletviewer
  • You are trying to use a stand-alone applet version of this page in an environment where jogl is not locally installed in the java extension directory
