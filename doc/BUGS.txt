Following a list of detected BUGS.
  - SimpleBody.doIntersection is not taking into account current body
    scale. Moreover, this method is apparently inducing errors on
    raytracing (probably on the normal managment). Compare reference
    raytracer implementation against original MIT sample or other raytracings.
    Method needs formal verification / reconstruction.
  - PolyhedralBoundedSolid.computeQuantitativeInvisibility seems to be
    failing. To repeat this error, the PolyhedralBoundedSolidExample with
    Appel algorithm for hidden line visualization can be used.
  - PolyhedralBoundedSolidSetOperator is still failing in some cases
    after the vertex/vertex classification. It appears that the setOpConnect
    procedure is not detecting all possible cases of orientation for
    neighbor half edges.
  - PolyhedralBoundedSolidSetOperator current detected failing cases:
    . PolyhedralBoundedSolidModelingTools.buildCsgTest5() on operations
      difference and intersection.
    . SimpleTestGeometryLibrary.createTestObjectPairMANT1988_15_2(); goes
      well up to connect, fails on finish and maximize faces.
  - JOGL/OpenGL related
    . On JoglView (SceneEditorApplication), tex labels renderered using the
      transparent image technique changes the colors wrongly when a scene
      with textured objects is displayed. This bug is appearing only
      on Windows Vista with intel videocards.
  - NVidia Cg related
    . Shader selection must be based on the current availability
      of maps (texture and normal maps), and not only on selected
      RendererConfiguration. This leads to incorrectly renderings when
      requesting for example bump mapping on an object without a loaded
      normal map.
    . Display lists freezes CG parameters to the values valid only for
      object when compiling the list, making object inalterable.
  - On testsuite/VSDKExamples/CgShaderExample CgAutomaticShaderExample.java
    test object fails to activate texture map on first frame.
  - Torus
