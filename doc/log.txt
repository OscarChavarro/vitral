===========================================================================
= VitralSDK simplified Personal Software Process file                     =
===========================================================================

This text file contains the main "Personal Software Process" (PSP) for the
VitralSDK project. Every contributor for VitralSDK is encourage to write
in this file its major acomplishments in a session basis. Note that every
entry in this file corresponds to a commit in the SVN control version system.

Rough format in this file is:
  - For each change, ther is a text line starting with the character '-',
    followed by the time taken and the description of the activity.
  - Each set of changes done the same day are grouped by a "title" in the
    format

YEAR month day, author1, author2 ... & author n: <<total sum of minutes>>

Note that this log include the main work done on core VitralSDK libraries and
"smaller" programs in testsuite folder. This PSP does not include work done
on "big" programs, as such:
  - ./testsuite/OfflineExamples/RaytracingOfflineExample
  - ./testsuite/ApplicationCases/SciVisApplication
  - ./testsuite/ApplicationCases/SceneEditorApplication
for that, there are other similar "log.txt" files with its own PSP logs.

---------------------------------------------------------------------------

2006 march 14, Oscar Chavarro, Eduardo Mendoza, Fabio Aroca: <<?>>
  - Library is now packaged in just one big dir, not using the old
    stages order directories. Now all the examples are inside the 
    testsuite directory.
  - Incorporated ZBuffer and JoglZBufferRenderer classes from Fabio Aroca
    & Eduardo Mendoza main work, allowing for distributed rendering based
    on ZBuffer algorithms and other manipulations for depth maps.
  - New organization of packages and classes inside packages. Below are
    the old names and the new names for packages:
    . vitral is no longer called such that, it is now "vsdk" which stands for
      "Vitral Software Development Kit".
    . Inside the VSDK architecture, the current toolkit is one, so the main
      package is called "toolkit" instead of "toolkits".
    . Package "geometry" is now inside package "environment"
    . Old package "visual" is now called "render"
    . Old package "util" is now called "io"
    . Package "io" is organized such as each data type has its own subdirectory
      so class ReaderObj is in io.geometry
    . Classes ImageNotRecognizedException, RGBAImageBuilder and TargaImage 
      which handles image loading from files were moved to package io.image.
  - The main library is not "vitral.jar", but "vsdk.jar"
  - As a cylinder is a special case of cone, class Cylinder is now called
    Cone
